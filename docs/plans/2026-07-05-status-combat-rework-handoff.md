## Part 1 — 現在必修的 bug / 漏洞

目前 12 屬性狀態系統（`src/main/java/me/yisang/limbusego/status/`）跑在伺服器正常傷害管線「外面」，導致下列問題。以下 file:line 為 2026-07-05 時的位置，可能有偏移，以函式名為準。

### 🔴 A. 領地內攻擊被取消、DoT 仍施加，且傷害繞過保護（可掛人）
- **位置**：`gift/GiftsModule.java` 的 `onEntityDamageByEntity`（約 line 633）。
- **問題**：`@EventHandler` 沒有 `ignoreCancelled`，也沒檢查傷害量。領地/保護插件（WorldGuard、Residence…）通常在低優先度 `cancel` 掉 PvP 事件，但這個 NORMAL 監聽照樣跑 → `onAttack` 照樣施加 DoT。
- **後果**：在禁 PvP 的領地裡打玩家，直接傷害是 0，但 DoT 照疊上去，然後（見 B）那個 DoT 傷害還繞過領地保護把人扣死。**這是可被利用的掛人漏洞。**
- **修正方向**：
  1. 改成 `@EventHandler(ignoreCancelled = true)`；
  2. dispatch 前加閘：`event.getFinalDamage() > 0` 才施加狀態。
  - 副帶好處：也一併堵掉「空手 1 點傷猛戳玩家免費疊 DoT」——因為零傷/被取消的攻擊不再施加狀態。

### 🔴 B. DoT 用裸真傷，繞過護甲 / 附魔 / 領地
- **位置**：`status/StatusManager.java` 的 `dealTrueDamage`（約 line 319）用 `target.damage(amount)`，**不帶來源**、且在排程 task 上執行。
- **問題**：這種傷害不觸發正常 PvP 判定（領地管不到）、也吃不到護甲與附魔。
- **修正方向**：改用**帶來源**的 `target.damage(amount, sourcePlayer)`，讓 DoT 走正常 `EntityDamageByEntity` → 領地、PvP 旗標、護甲、附魔**自動全部生效**。
  - 保留現有的 `CUSTOM_DAMAGE_META` 防遞迴標記。
  - 若想保留一部分穿透感，可只讓「一部分比例」走真傷、其餘走正常傷害（見 Part 3 的「地板」）。

### 🔴 C. PROTECTION / FRAGILE 永久累積 → 無敵 / 秒殺
- **位置**：`status/StatusManager.java` 的 `onDamage`，PROTECTION（約 225）、FRAGILE（約 231）兩段。
- **問題**：這兩個屬性在傷害事件裡**只讀 potency 套乘區、從不 `consume`**；再加上 `StatusState.add()`（約 line 16）讓 potency **只增不減**（只有 count 會衰減）→ potency 永久累積、無上限。
  - `PROTECTION` 是 `×(1 − 0.05 × potency)`，程式無封頂 → potency 疊到 20 就是**減傷 100% = 無敵**。
  - `金剛丸` / `靜蓮丸` 每 5 秒（onPassiveTick）補 PROT → 站著約 50 秒就無敵。
  - `FRAGILE` 是 `×(1 + 0.15 × potency)` 同理可疊到秒殺線。
- **受影響飾品**：
  - PROT：`JinGangBolus`、`TranquilLotusBolus`（皆 onPassiveTick 每 5 秒補）、`TenacityBolus`、`LateBloomersTattoo`、`TheBookOfVengeance`。
  - FRAG：`HandheldMirror`、`GlimpseOfFlames`、`SpecialContract`、`Ruin`。
- **修正方向**（擇一或並用）：
  1. 在 `StatusState.add()` 給每個 effect 的 potency 設**上限夾住**（順便也治下面 D 的「潛在」爆表）；
  2. 或讓 PROT/FRAG 也隨受擊 `consume` count，不再永久留存。

### 🟡 D. （順手）potency 平方疊加
- 因為 potency 只增不減，反覆施加同一 DoT 會讓「每次觸發傷害 × 觸發次數」呈平方成長（例：火光花 3 次命中 ≈ 36 真傷）。若不做 Part 3 的大改，至少在 `add()` 夾住 potency 上限即可止血。

## Part 2 — 根本病因（一句話）

> **整個狀態系統跑在伺服器真正的戰鬥管線「外面」。** 它施加時不管攻擊合不合法、有沒有真的造成傷害；結算時用裸真傷繞過護甲/附魔/領地。

上面 A~D 全是這一件事的不同臉。把「施加」與「結算」都拉回正常傷害管線，就同時解掉領地漏洞、附魔需求、和「PvP 傷害無從約束」的問題。這也正是 Part 3 想做的事。

## Part 3 — 未來方向：還原 Limbus 的抗性戰鬥模型

### 核心論點
原作者把 **Limbus 本身已經平衡好的戰鬥模型丟掉了**，換成一堆手填的真傷 —— 這就是「數值亂填、不可量化」的真正病因。Limbus 的傷害之所以能平衡，靠的是**型別 × 罪的抗性框架**在約束；插件把框架拔掉、每顆飾品各填各的真傷，當然失衡。

**要做的不是設計新抗性，是把源作已經被官方 tune 過的系統搬回來。**

### 模型
每次攻擊帶兩個標籤：**物理型別（斬 / 突 / 打）** 與 **七大罪之一**。每個實體對這兩軸各有 affinity（抗性倍率），**範圍 0.4 ~ 2.0、以 1 為中心**（1 = 中性、>1 挨得多、<1 挨得少）。兩軸**相乘**再套到傷害：

```
最終傷害 = 基礎傷害 × 型別抗性 × 罪抗性 ×（原版附魔/護甲，因為已走正常管線）
例：武器「斬 · 憂鬱 · 5」打 目標(斬×1.2, 憂鬱×0.8) → 5 × 1.2 × 0.8 = 4.8
```

- **離散 → 連續**：Limbus 是離散抗性階（×0.5/×1/×2），不適合 MC 裝備自訂，所以攤成連續值。
- **增傷 buff 用加法**：多個 +30% = +90%（×1.9），不是連乘（避免乘積失控）。
- **地板**：每維洗鍊**不低於 0.4** → 雙維最低 `0.4×0.4 = 0.16`，永遠咬得到 16% 傷害，**不可能完全免疫**。

### DoT 併入這個模型（治「破裂封 90 還是暴斃」）
四條 DoT（燒 / 血 / 裂 / 顫）改法：
1. **走正常傷害管線**（Part 1-B）→ 吃得到抗性/附魔。
2. **每層線性、固定基礎傷害**，丟掉現在的 `potency × 倍率` → 平方疊加病根消失。
3. **層數（count）上限 90、且隨時間自然衰減**（硬需求：跑開/喝奶可以拖掉）。
4. **DoT 只吃「罪」這一維**（燒傷沒有斬突打）→ 地板是**單維 0.4**（不是雙維 0.16）。這是白賺的好性質：**DoT 天生比直擊更難完全扛住**，「持續傷害賴著不走」的手感從數學結構自然掉出來。
- 「每一下多痛」由抗性+附魔+地板管；「總共持續多久」由 90 上限+衰減管。兩者搭配才不會「90 層 = 死刑」。

## Part 4 — 裝備 / 洗鍊 / 材料（未來）

抗性從哪來：玩家預設全 1，靠**四件裝備**各自帶 +/- delta 疊加而成。

```
finalResist[維] = clamp( 1 + Σ(四件的 delta), 0.4, 2.0 )   // 夾在「加總後」，不是每件
```
> 「四件相加 + 每件機率貼近 0」＝ 中央極限定理 → 總抗性自然收成鐘形、峰在 1，**極端組合天生稀有**，不太需要靠硬 clip。

### 物理抗性的零和 score（靈魂機制）
物理三維（斬突打）**每件必出**，用分數約束成零和取捨 → **不可能物理全抗**。

- 記號：`d = affinity − 1`（d<0 抗、d>0 弱）。
- 成本（吃錨點「斬 −0.2 花 10 分」）：
  - 抗性側 `d<0`：花費 `= 50 × |d|`（−0.2 → 10 分）
  - 弱點側 `d>0`：退回 `= 25 × d`（+0.4 → 退 10 分）→ **2:1 不對稱**：變抗比變弱貴一倍。
  - 建議升級：抗性側改凸函數 `50|d| + C·d²`，越接近 0.4 越貴 → 地板從「成本」自然長出來、不用硬 clip。
- **材料等級 → 基礎分數 `B(T)`**，一件裝備的 roll 滿足 `Σ cost(d) = B(T)`：
  - `B = 0` → 純重分配、均值釘在 1（白板/低階材料）。
  - `B > 0` → 淨抗預算、均值 < 1 = **真的變肉**（高階材料）。
  - **`B(T)` 這條曲線幾乎握著整個物理防禦的強度**，是唯一最該謹慎手 tune 的東西：先想「頂裝對主要傷害類型該減多少%」，反推每階 B。
- 三維先**全部等權**；等看到伺服器上哪種傷害佔比高再加權（太早加權＝重蹈「憑感覺填」覆轍）。

### 深度專精的代價（＝反制引擎）
四件全堆斬抗 → 打擊維可能被推到 **2.0（被打雙倍，clamp 封頂於此，不會更慘）**。這是**刻意的**：
- PvP 對面發現你堆斬抗 → 換打擊武器打穿你；PvE 斬 tank 進打擊怪區 → 寸步難行。
- 這就是 Limbus 的核心（配型別/罪剋弱點），也自然懲罰無腦 min-max。

### 洗鍊採「半指定」
隨機決定**數值分布**，但玩家能**鎖定/保護某一維**或選一個**主抗方向** → 有 loot 隨機爽感，又不會洗出「對最常見傷害 ×2」的自殺裝。

### 材料設計（各司其職）
| 材料 | 職責 | 稀有度原則 |
|---|---|---|
| **抗性精華（分階）** | 決定 `B(T)` 淨抗預算（多強） | **稀有**：進度/強怪/boss 門檻，慢慢給 |
| **導向試劑（斬/突/打/罪）** | 半指定「往哪維」 | **便宜、可重複**：可合成/可農 |
| **鎖定媒介** | 洗前保護一維不動 | 中等（可做成試劑高階版，收成兩種材料）|

> **原則：把「強度」做稀有，把「方向」做便宜。** 貴的是天花板、不是實驗。換方向要幾乎免費，型別剋制的玩法才活得起來；否則大家鎖死一套 build，反制玩法直接死掉。

## 附錄 — 白話解釋區（給作者，不用再問人）

**Q：什麼是「真傷」，為什麼不好？**
現在 DoT 用的是「無視一切的傷害」——不管你穿什麼甲、有沒有保護附魔、在不在禁 PvP 領地，它就是直接扣血。好處是數字好預測，壞處是它**跳過整個遊戲的防禦系統**：甲白穿了、領地保護形同虛設。把它改成「正常傷害」，甲和領地就自動管得到它。

**Q：affinity 抗性（0.4~2.0）是什麼意思？**
一個乘在傷害上的倍率。1 = 正常；0.8 = 只吃八成（你抗這種傷害）；1.2 = 吃一點二倍（你怕這種傷害）。斬突打一個倍率、七大罪一個倍率，兩個乘起來。所以「抗斬但怕打擊」的人，被斬砍軟綿綿、被鈍器打會很痛。

**Q：為什麼增傷用加法、抗性用乘法？**
增傷用加法（+30%+30% = +90%）是為了**不讓它連乘爆炸**。抗性用乘法（型別 × 罪）是因為那本來就是兩個獨立的維度，各自縮放傷害，相乘才對。

**Q：為什麼「不能物理全抗」是好事？**
如果玩家能同時抗斬、抗突、抗打，那就無敵了、沒人打得動。零和 score 逼你「選一個方向硬、另一個方向就會軟」。這樣每套 build 都有弱點可以被剋，戰鬥才有攻防、有策略，而不是比誰數字堆得高。

**Q：為什麼 DoT 要「層數會衰減」？**
MC 一條命才 20 血。任何能疊很多層的持續傷害，如果不會自己消退，疊滿了就是「保證死」。讓層數隨時間掉，玩家才有「跑開、喝罐奶、等它退」的活路。

**Q：這一切要一次做完嗎？**
不用。**Part 1 的三個漏洞先修**（那是會壞遊戲的），其餘（抗性模型、洗鍊、材料）是可以分好幾期慢慢長的願景。架構上先把十個抗性維度的欄位開好、DoT 拉回正常傷害管線，之後填數值就好。
```
